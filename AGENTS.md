# Agent Rules

## Build and Test Verification (Repo-Specific)

ALWAYS verify that the package builds successfully and tests pass after making changes:

1. Run the package build: `yarn turbo run build --filter=@kirill.konshin/<package-name>`
2. Run the package tests: `yarn turbo run test --filter=@kirill.konshin/<package-name>`
3. You may use grep or similar filtering (e.g., `vitest run -t "pattern"`) to limit tests when you are confident the filter covers all relevant tests
4. If in doubt about test coverage, the final verification step should be to run all tests in the affected package

Do not consider work complete until build and tests pass.

# TypeScript

All packages extend root `tsconfig.json`.

## Compiler Options

Key TypeScript settings used across the codebase:

- `isolatedDeclarations: true` - Enables faster declaration emit and requires explicit type annotations on exports
- `strictNullChecks: true` - Null and undefined are distinct types
- `strict: false` - Not using full strict mode, but strictNullChecks is enabled
- `moduleResolution: "bundler"` - Modern bundler-compatible resolution
- `jsx: "react-jsx"` - Modern JSX transform (no React import needed)

# Dev Dependencies

Repo is using hybrid NX-like dependency management strategy:

- [Single Version Policy](https://nx.dev/concepts/decisions/dependency-management#single-version-policy) for truly common packages like `react`, `typescript`, `wait-on` are defined in root `package.json`
- [Independently Maintained Dependencies](https://nx.dev/concepts/decisions/dependency-management#independently-maintained-dependencies) for all other packages
- Except demos, all dependencies are defined individually in their `package.json`

# Package Configuration Standards

All packages in `packages/` (except `utils-private` and `eslint-config-next-custom`) follow identical configuration patterns.

# Required Files Per Package

Each package MUST have these files with standardized content:

## `vite.config.ts`

```ts
import config from '../utils-private/vite.config';
export default config;
```

## `tsconfig.json`

```json
{
    "extends": "../utils-private/tsconfig.json",
    "compilerOptions": {
        "rootDir": "src",
        "outDir": "dist",
        "declarationDir": "dist"
    },
    "include": ["src"],
    "exclude": []
}
```

## `turbo.json`

```json
{
    "$schema": "https://turbo.build/schema.json",
    "extends": ["//"],
    "tasks": {
        "build": {
            "dependsOn": ["^build"],
            "outputs": ["src/**/*/index.ts", "package.json", "dist/**/*"]
        }
    }
}
```

## `package.json` Scripts Section

All packages use identical scripts (copy exactly):

```json
{
    "scripts": {
        "----- BUILD -----": "",
        "clean": "rm -rf dist .tscache tsconfig.tsbuildinfo",
        "build": "vite build",
        "build:index": "cti create ./src",
        "build:check-types": "attw --pack .",
        "start": "yarn build --watch",
        "wait": "wait-on ./dist/index.js",
        "----- TEST -----": "",
        "test": "vitest run --coverage",
        "test:watch": "vitest watch --coverage",
        "----- STORYBOOK -----": "",
        "storybook:start": "storybook dev -p 6006",
        "storybook:build": "storybook build"
    }
}
```

Note: Use `"test:disabled"` instead of `"test"` if tests are not yet implemented.

## `package.json` Structure Rules

1. **Required fields**: `name`, `description`, `version`, `type: "module"`, `scripts`, `publishConfig`, `author`, `license`
2. **Exports format** (always use this exact structure):

```json
{
    "exports": {
        ".": {
            "import": {
                "import": "./dist/index.js",
                "types": "./dist/index.d.ts"
            }
        }
    },
    "main": "./dist/index.js",
    "module": "./dist/index.js",
    "types": "./dist/index.d.ts"
}
```

3. **DevDependencies**: Always include `"@kirill.konshin/utils-private": "*"`
4. **PeerDependencies**: Framework deps (react, next, electron, etc.) go in `peerDependencies` with `optional: true` in `peerDependenciesMeta`
5. **Dependencies**: Only runtime dependencies that consumers need

## Auto-Generated Fields (DO NOT manually edit)

The following `package.json` fields are auto-generated by `packages/utils-private/vite.exports.ts` during build via `fixExports()`:

- `exports` - generated from `src/*/` folder structure
- `main`, `module`, `types` - derived from main export
- `type` - set to `"module"`
- `publishConfig` - set to `{ "access": "public" }`
- `author`, `license` - standardized values
- `scripts.wait` - generated from main export path
- `peerDependenciesMeta` - auto-populated with `optional: true` for all `peerDependencies`
- `peerDependencies` version ranges - normalized from `devDependencies` versions

**Only manually set**: `name`, `description`, `version`, `dependencies`, `devDependencies`, `peerDependencies` (keys only, versions are normalized)

# Creating a New Package

1. Copy an existing package folder (e.g., `packages/react`)
2. Update `name` in `package.json` to `@kirill.konshin/<package-name>`
3. Keep all config files (`vite.config.ts`, `tsconfig.json`, `turbo.json`) unchanged
4. Adjust dependencies as needed
5. Create `src/index.ts` as the entry point

# Source File Organization

## Index Files

Use barrel exports in `src/index.ts`:

```ts
export * from './cache';
export * from './errors';
export * from './files';
export * from './measure';
```

## .ctirc Configuration

Each package has a `.ctirc` file for `create-ts-index`:

```json
{
    "addNewline": true,
    "fileExcludePatterns": ["*.stories.*", "*.test.*", "*.fixture.*"],
    "verbose": false,
    "withoutBackupFile": true,
    "withoutComment": true
}
```

This excludes test, story, and fixture files from the generated index.

# Turbo

All common library tasks are defined in root `turbo.json`: `clean`, `build`, `start` & `wait`.

Library packages run `start` (`vite build --watch`) which produces `dist/index.js`.
